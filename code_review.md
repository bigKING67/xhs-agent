# code_review.md

本文件定义本仓库的代码评审最小标准，优先关注缺陷、回归风险与验证覆盖。

## 评审优先级
- `P0` 阻断级：会导致错误结果、数据破坏、安全问题或不可用。
- `P1` 高优先级：明显功能缺陷、兼容性问题、可维护性风险较高。
- `P2` 一般问题：风格、可读性、小型优化建议。

## 核心检查清单

### 1) Correctness
- 需求是否被完整实现。
- 边界条件是否处理（空值、超时、重试、异常路径）。
- 是否引入静默降级、吞错、假成功。

### 2) Regression Risk
- 是否影响现有命令/接口兼容性。
- 是否改动了与需求无关的模块。
- 是否有回归测试覆盖关键路径。

### 3) Security
- 是否硬编码密钥、Token、Cookie 或敏感信息。
- 外部输入是否有边界校验与清洗。
- 危险操作是否有显式确认机制。

### 4) Test & Verification
- 是否执行了最相关的最小验证命令。
- 新行为是否有测试或可复现实验命令。
- 未验证项是否明确记录。

### 5) Maintainability
- 命名和结构是否清晰。
- 逻辑是否聚焦根因修复。
- 注释是否简洁且解释复杂部分。

## 评审输出模板

```markdown
## Findings
1. [P0] <file:line> <问题描述>
2. [P1] <file:line> <问题描述>
3. [P2] <file:line> <建议>

## Open Questions / Assumptions
1. <待确认问题或假设>

## Verification
- 执行: `<command>`
- 结果: <pass/fail/未执行>

## Summary
- 改动范围: <files/modules>
- 残余风险: <risk>
```

## 通过标准
- 不存在未处理的 `P0`。
- `P1` 要么修复，要么给出明确风险说明和后续处理计划。
- 验证结果可复现，且与改动范围匹配。
